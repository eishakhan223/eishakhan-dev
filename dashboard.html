💡 Key Insights
📅
Seasonal Peaks Detected
March/April (Eid) and December/January (Winter) consistently show 15–25% sales uplift across all SKUs.
🏷️
Promotions Drive +20% Lift
Weeks with active promotions generate a statistically significant 20% average demand spike — a critical feature for the model.
🔮
Lag Features Are Key
lag_1m (previous month sales) contributes the highest feature importance in Random Forest, confirming autocorrelation in demand.
📊
Linear Model Wins on MAPE
Despite its simplicity, Linear Regression achieved the best MAPE (10.68%) — suggesting the demand signal is largely linear after feature engineering.
🏪
Uniform Store Distribution
All 4 stores contribute roughly equally (~25% each), indicating no single store dominates — good for generalizable model training.
⚠️
Overstocking Risk — Mid-Year
Jul–Sep consistently shows demand troughs. Inventory buffers should be reduced by 10–15% in Q3 to minimise holding costs.
🔧 Technical Pipeline

① Data Preprocessing

  • 6,656 weekly raw records generated
  • Aggregated to 1,536 monthly rows
  • Missing values imputed (median)
  • Chronological sorting enforced
  • Date parsed to datetime format

② Feature Engineering

  • Lag features: 1, 2, 3, 6, 12 months
  • Rolling mean: 3m, 6m, 12m windows
  • Rolling std: 3m window
  • Month, quarter, year, sin/cos
  • Promotion flag & avg price

③ Evaluation Strategy

  • Temporal train/test split
  • Last 6 months reserved for test
  • MAE — average unit error
  • RMSE — penalises large errors
  • MAPE — business-readable %
